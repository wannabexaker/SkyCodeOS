## Architecture Guardrails (Strict — Never Violate)

### Layer Stack (Top → Bottom)
```
Models
  ↓ [adapter only]
Inference Runtime (llama.cpp + optional remote adapter)
  ↓ [SkyCore Protocol only]
Orchestrator (policy engine, approval gate, task routing)
  ↓
Agent Runtime / Memory / Graph / Tools (orchestrator only — no direct model calls)
  ↓
CLI (v1 interface — binary only)
```

### Forbidden Crossings — Hard Errors
- **Agent Runtime calls llama.cpp directly** → FORBIDDEN
- **Any file write without valid ApprovalToken** → FORBIDDEN  
  - Token: UUID v4, TTL=300s, single-use via atomic INSERT
  - 13-step validation + replay defense (docs/protocol.md)
- **CLI receives provider-format response (OpenAI/llama shapes)** → FORBIDDEN
- **UPDATE or DELETE on: tool_events, approval_tokens_used, applied_changes** → FORBIDDEN
  - These tables have SQLite triggers that raise errors

## Code Rules (Absolute)

### Rust Edition & Toolchain
- Edition: **2021**
- Toolchain: **stable**
- Pins: See `PINS.yaml` (if created during project setup)

### Error Handling
- All custom errors **derive `thiserror::Error`**
- All functions that can fail return **`Result<T, E>`**
- **No `unwrap()`** outside `#[cfg(test)]` blocks
- **No `expect()`** in production paths
- Errors propagate via `?` operator

### SQL & Database
- All queries via **`rusqlite` prepared statements**
- **Never string interpolation with user data** (SQL injection risk)
- All schema structures defined in `docs/schemas.md`
- Append-only tables enforced by SQLite triggers — never write UPDATE/DELETE

### Dependencies (Required — No Substitutions)
```toml
rusqlite = { version = "0.31", features = ["bundled"] }
thiserror = "1"
uuid = { version = "1", features = ["v4"] }
serde = { version = "1", features = ["derive"] }
serde_json = "1"
ring = "0.17"              # ed25519 signing for approval tokens
tree-sitter = "0.22"       # code graph parsing
clap = { version = "4", features = ["derive"] }  # CLI argument parsing
```

## Phase Order (Sequential — No Skipping)

### Phase 1: Safe Tool Spine
**Goal:** Read/diff/approve/apply/rollback + immutable audit log

1. `runtime/src/tools/mod.rs` — Tool trait definition
2. `runtime/src/tools/filesystem.rs` — read_file, list_dir, search_project, git_status (read-only)
3. `runtime/src/tools/diff.rs` — create_diff → DiffProposal struct
4. `runtime/src/approval/token.rs` — ApprovalToken struct, creation, serialization
5. `runtime/src/approval/validator.rs` — validate_token() with 13-step check + atomic INSERT
6. `runtime/src/tools/apply.rs` — apply_diff() (requires valid token from validator)
7. `runtime/src/tools/rollback.rs` — rollback() reverts to pre-apply git state
8. `runtime/src/db/events.rs` — tool_events INSERT (append-only logger)
9. `runtime/src/db/migrations.rs` — run migrations from `memory/migrations/`
10. CLI commands: `diff`, `approve`, `apply`, `rollback`

**Exit Gate:** 50 simulated edit cycles with zero unapproved writes in audit log.

### Phase 2: Memory + Graph
**Goal:** SQLite FTS5 retrieval + tree-sitter dependency indexing

1. `memory/migrations/001_initial.sql` — all 13 tables from docs/schemas.md
2. `runtime/src/memory/store.rs` — INSERT memories + memories_fts
3. `runtime/src/memory/retrieval.rs` — FTS5 search with BM25 + recency + importance ranking
4. `runtime/src/graph/indexer.rs` — tree-sitter walker → graph_nodes + graph_edges
5. `runtime/src/graph/impact.rs` — recursive CTE impact query (what files depend on this symbol?)
6. CLI command: `graph impact <symbol>`

**Exit Gate:** Impact query correctly identifies affected files on a real codebase.

### Phase 3: Local Inference + SkyCore Protocol
**Goal:** llama.cpp integration + provider-agnostic protocol boundary

1. `runtime/src/inference/loader.rs` — GGUF model loading via llama.cpp
2. `runtime/src/inference/context.rs` — context window management
3. `runtime/src/inference/registry.rs` — model registry YAML loader
4. `runtime/src/skycore/request.rs` — SkyCore request struct + serialization
5. `runtime/src/skycore/response.rs` — SkyCore response struct + deserialization
6. `runtime/src/skycore/boundary.rs` — strips provider-specific fields before CLI

**Exit Gate:** Local GGUF model completes a SkyCore round-trip with **network disabled**.

### Phase 4: Persistent Coder Agent
**Goal:** Full task loop, session continuity, doctrine enforcement

1. `runtime/src/agent/identity.rs` — load soul/heart/mind/doctrine from YAML
2. `runtime/src/agent/state.rs` — agent_state table persistence
3. `runtime/src/orchestrator/task_loop.rs` — classify → context → invoke → diff → approve → apply → log
4. `runtime/src/orchestrator/policy.rs` — doctrine enforcement, approval gate logic
5. CLI command: `ask "<task>"`

**Exit Gate:** Agent recalls a decision from session 1 in session 3 after restart. One real edit offline end-to-end.

### Phase 5: Hardening + Router + Testing Lab
**Goal:** 100+ named tests, context budget enforcement, model router

1. All tests from `docs/testing.md` (100+ named tests)
2. Context budget enforcement (docs/context-budget.md — ≥50% reduction required)
3. Model router — task classifier → registry lookup → fallback chain
4. Runtime tuning profiles (fast/deep/precise/creative per docs/profiles.md)
5. Regression tests — all Phase 1–4 exit gates re-verified

**Exit Gate:** Full regression suite passes. Zero unapproved writes across all scenarios.

- Cargo: `cargo publish`, `cargo install`

- Anything that modifies files outside the workspace

- Web fetching for architecture decisions (use local docs/)
- Async tools for Phase 1–2 (unless llama.cpp streaming requires it)
- Multi-file simultaneous editing for approval-critical code

## Non-Negotiables

These are absolute constraints. Never violate them, even if it seems convenient:

1. **No silent writes.** Every file mutation requires a validated `ApprovalToken` (UUID, TTL=300s, single-use atomic INSERT).
2. **tool_events is append-only.** No UPDATE, no DELETE, no exceptions. Enforced by SQLite trigger.
3. **All Phase 1–4 exit gates must pass with network disabled.**
4. **No full-file context dumps.** Only graph + memory retrieval via ranking.
5. **Agent never bypasses orchestrator** for tools or model access.
6. **SQLite FTS5 only** until retrieval quality measurably fails (with benchmark).
7. **Single agent (coder-primary) in V1** — no multi-agent code yet.
